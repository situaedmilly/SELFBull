# SELFQUANT ↔ SELFBull Integration Plan — DESIGN ONLY (Phase 8)

> **Status: DESIGN ONLY.** Nothing in this document is implemented. SELFQUANT
> was not mutated to produce this plan — it lives entirely in SELFBull's own
> `docs/`, describing SELFBull's side of a future exchange. No code in
> either repository implements what follows yet.

## The future exchange

```
SELFQUANT                                   SELFBull
  governs an intent via                       validates the JSON schema
  governed_decision()                         records a receipt
  emits a PreparedOrderIntent (JSON)  ────►    returns a BrokerTransportReceipt
                                               mode: Phase 1 = blocked/simulated only
```

SELFQUANT emits an already-governed, broker-neutral `PreparedOrderIntent`
(schema in `docs/SELFBULL-INTERFACE-CONTRACT.md §4`) as JSON — never a
Python object, never a shared class. SELFBull:

1. Validates the JSON against the `PreparedOrderIntent` schema.
2. Records the receipt (audit-safe, redacted).
3. Returns a `BrokerTransportReceipt` — during Phase 1, always a refusal.

SELFBull does not re-judge the intent. It never asks "should this trade
happen" — that question was already answered by SELFQUANT before the JSON
crossed the wire. SELFBull only asks "can I attempt transport," and in
Phase 1 the answer is structurally always no.

## `BrokerTransportReceipt`

| Field | Type | Notes |
|---|---|---|
| `schema_version` | string | `"1.0"` |
| `receipt_id` | string | locally generated by SELFBull |
| `intent_id` | string | echoes the `PreparedOrderIntent.intent_id` received |
| `broker` | string | `"webull"` |
| `received_at` | string | ISO-8601 UTC |
| `mode` | string | `"phase1_inert"` (future: `"phase2_read_gated"`, etc.) |
| `accepted_for_transport` | bool | **always `false` in Phase 1** |
| `transmitted` | bool | **always `false` in Phase 1** |
| `broker_order_id` | string\|null | **always `null` in Phase 1** |
| `refusal_code` | string\|null | e.g. `"NO_TRANSPORT_IMPLEMENTATION"`, `"KILL_SWITCH_ACTIVE"` |
| `refusal_reason` | string\|null | human-readable |
| `evidence` | object | provenance |

### Phase-1 invariant (repeated because it is load-bearing)

- `accepted_for_transport = false`
- `transmitted = false`
- `broker_order_id = null`

These three fields cannot be anything else in Phase 1 — there is no
transport implementation to accept, transmit, or assign a broker id.

## What SELFQUANT would need to change (future, not now)

- A JSON serializer for `PreparedOrderIntent` matching the schema above,
  populated from `governed_decision()`'s output plus order-shape fields.
- A thin HTTP or file-drop client to hand that JSON to SELFBull — never a
  Python import of `selfbull.*`.
- A deserializer for `BrokerTransportReceipt` to fold back into SELFQUANT's
  own evidence plane (`agent_decision_trace.jsonl` or a sibling log).

None of this exists yet, and building it is out of scope for this pass —
SELFQUANT was not mutated to write this document, and won't be mutated to
implement it without a separate, explicit authorization.

## What SELFBull would need to change (future, not now)

- A JSON schema validator for the four inbound/outbound envelope shapes.
- A `BrokerTransportReceipt` constructor mirroring `PreparedOrderIntent`'s
  pattern (frozen dataclass, `accepted_for_transport`/`transmitted` not
  settable to `true` via any public constructor argument in Phase 1 — same
  structural-refusal technique already used for `execution_authority`).
- Whatever real transport channel (HTTP endpoint, file drop, message queue)
  the two repos agree carries the JSON — not decided here.

## Non-goals of this document

- Does not choose a transport mechanism (HTTP vs. file vs. queue) — that is
  a separate, later decision.
- Does not implement anything in either repository.
- Does not change SELFQUANT's `BROKER_LIVE_ROUTE_AVAILABLE` flag or gate
  ladder in any way.
